# Contributing to Payloadify

Thanks for your interest in contributing!

## How to Contribute

1. Fork the repo
2. Create a branch (`git checkout -b feature/your-feature`)
3. Make changes
4. Test locally (`npm run dev`)
5. Commit with clear messages
6. Open a PR describing what changed

## Guidelines

- Keep tool outputs **spec-compliant** (test against official specs/RFCs)
- Write clear error messages (security professionals read them)
- Avoid external dependencies if possible (keep bundle size low)
- Follow existing code style
- Run `npm run lint` and `npm run test` before opening a PR

## Reporting Bugs

Open an issue with:

- What tool/feature
- Expected behavior
- Actual behavior
- Steps to reproduce

Thanks!
